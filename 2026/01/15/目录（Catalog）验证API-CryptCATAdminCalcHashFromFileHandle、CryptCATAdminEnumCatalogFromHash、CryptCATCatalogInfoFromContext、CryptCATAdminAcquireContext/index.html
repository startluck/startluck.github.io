目录（Catalog）验证API-CryptCATAdminCalcHashFromFileHandle、CryptCATAdminEnumCatalogFromHash、CryptCATCatalogInfoFromContext、CryptCATAdminAcquireContext

发表于2026-01-15|更新于2026-01-15
|浏览量:

CryptCATAdminAcquireContext

基本概念

CryptCATAdminAcquireContext是Windows 目录（Catalog）验证体系的 “入口函数” —— 所有目录相关操作（计算文件哈希、查找目录条目、提取.cat 文件路径）都必须先通过这个函数获取目录管理上下文句柄（HCATADMIN），它的作用类似于打开一个 “目录操作的会话”。

基本语法

返回值
TRUE-成功
FALSE-失败

  1. ERROR_OUTOFMEMORY：内存不足
  2. ERROR_INVALID_PARAMETER：参数未初始化（如phCatAdmin没设NULL）
  3. ERROR_ACCESS_DENIED：权限不足（需管理员）

CryptCATAdminCalcHashFromFileHandle

基本概念

这个函数是Windows 目录（Catalog）验证体系的核心，作用是从已打开的文件句柄计算文件的哈希值（默认SHA-1），用于：
匹配系统目录文件（.cat）中的文件哈希条目；
间接验证文件的可信性（目录文件通过数字签名保证哈希的真实性）；
它是 “嵌入式签名验证失败后，兜底验证目录签名” 的关键步骤。

基本语法

返回值
TRUE：哈希计算成功；
FALSE：失败，可通过GetLastError()获取错误码（如ERROR_INSUFFICIENT_BUFFER表示缓冲区太小）。

CryptCATAdminEnumCatalogFromHash

基本概念

这个函数是Windows 目录（Catalog）验证的核心匹配函数，作用是：
基于文件的二进制哈希值，在系统的目录数据库（C:\Windows\System32\CatRoot）中查找对应的目录条目（HCATINFO） —— 每个目录条目关联一个.cat目录文件，而目录文件本身带有数字签名，验证这个签名就能间接确认文件的可信性。

基本语法

返回值
非 NULL：成功找到匹配的目录条目，返回HCATINFO句柄（需后续释放）；
NULL：未找到匹配条目，或调用失败（可通过GetLastError()排查，如ERROR_NO_MORE_ITEMS表示无更多条目）。

CryptCATCatalogInfoFromContext

基本概念

这个函数是Windows 目录（Catalog）验证的桥梁函数，核心作用是：
从CryptCATAdminEnumCatalogFromHash返回的目录条目句柄（HCATINFO）中，提取该条目对应的目录文件（.cat）路径—— 这是后续调用WinVerifyTrust验证目录签名的关键（因为验证目录签名需要知道.cat文件的具体位置）。

基本语法

数据类型
CATALOG_INFO:

返回值
TRUE：成功提取目录信息（wszCatalogFile会被填充有效路径）；
FALSE：失败，可通过GetLastError()获取错误码（如ERROR_INVALID_PARAMETER表示参数错误，ERROR_NOT_ENOUGH_MEMORY表示内存不足）。

简单示例

文件路径 → 获取目录上下文 → 计算文件哈希 → 查找目录条目 → 提取.cat 文件路径 → 验证目录签名

“配对释放” 原则：有获取必有释放
CryptCATAdminAcquireContext → 必须调用CryptCATAdminReleaseContext；
CryptCATAdminEnumCatalogFromHash → 必须调用CryptCATAdminReleaseCatalogContext；
