RegEnumKey

基本概念

RegEnumKey是Windows注册表API中的核心函数，用于枚举指定注册表项的子项名称（仅获取子项名称，不包含键值数据）。它属于早期注册表API（兼容 Windows 9x/NT），现代更推荐使用功能更强的RegEnumKeyEx（支持获取子项创建时间等额外信息），但RegEnumKey仍可在所有Windows版本中使用。

基本语法

返回值
返回值常量
ERROR_SUCCESS 枚举成功，lpName中存储有效子项名称。
ERROR_NO_MORE_ITEMS 所有子项已枚举完毕，无更多子项。
ERROR_MORE_DATA 缓冲区大小不足（cchName 太小），需增大缓冲区后重试。
ERROR_INVALID_HANDLE hKey是无效句柄（如未打开、已关闭或权限不足）。
ERROR_ACCESS_DENIED 调用进程无KEY_ENUMERATE_SUB_KEYS权限，无法枚举子项。

具体示例

枚举HKEY_CURRENT_USER\Software下的所有子项名称

RegEnumKeyEx

基本概念

RegEnumKeyEx是Windows注册表API的增强版子项枚举函数，继承了RegEnumKey的核心功能（枚举子项名称），并新增了获取子项元数据的能力（如创建时间、最后写入时间、子项类别等）。它是现代Windows应用的首选枚举函数，兼容Windows 2000及以上所有版本（替代老旧的RegEnumKey）。

基本语法

数据类型
LSTATUS：
LSTATUS是Windows注册表API（如RegOpenKeyEx、RegEnumKeyEx、RegQueryValueEx等）中专用的返回值类型，本质是一个32位有符号整数（LONG的别名），用于表示API调用的执行结果（成功、失败及具体错误类型）。

PFILETIME:

PFILETIME是Windows系统中指向FILETIME结构体的指针类型，核心用于在API函数中传递或接收 “文件时间” 数据（如注册表子项最后修改时间、文件创建时间等）。它是注册表API（如 RegEnumKeyEx）、文件操作 API（如GetFileTime）等的常用参数类型。
返回值
返回值常量
ERROR_SUCCESS 枚举成功，所有输出参数（lpName、lpClass、lpftLastWriteTime）有效
ERROR_NO_MORE_ITEMS 所有子项已枚举完毕，无更多子项
ERROR_MORE_DATA 缓冲区不足（lpName或lpClass大小不够），需增大对应缓冲区后重试当前索引
ERROR_INVALID_HANDLE hKey 是无效句柄（未打开、已关闭或权限不足）
ERROR_ACCESS_DENIED 进程无KEY_ENUMERATE_SUB_KEYS权限
ERROR_INVALID_PARAMETER 参数错误（如lpReserved非NULL、lpClass与lpcchClass不匹配）

具体示例

枚举HKEY_CURRENT_USER\Software下的子项，获取子项名称、类别、最后写入时间
