RegOpenKey

基本概述

RegOpenKey是Windows注册表API中用于打开已存在注册表项的简化版函数，核心作用是通过 “父键 + 子项路径” 定位已有的注册表项，并获取其HKEY句柄（后续用于读 / 写等操作）。它的特点是参数简单、默认权限，适合无需自定义权限或高级配置的基础场景。
注意：RegOpenKey仅能打开 “已存在” 的项，若项不存在会直接失败。

基本语法

返回值
成功：ERROR_SUCCESS（值为0）；
失败：非0错误码，常见如下：
错误码
ERROR_FILE_NOT_FOUND 目标子项不存在（核心区别于RegCreateKey）
ERROR_ACCESS_DENIED 权限不足（如打开HKLM下的项并尝试写入，未以管理员身份运行）
ERROR_INVALID_HANDLE hKey是无效句柄
ERROR_BAD_PATHNAME lpSubKey路径格式无效（含非法字符）

具体示例

用RegOpenKey打开HKCU\Software\MyApp项（需先确保该项已存在），读取AppVersion字符串值

RegOpenKeyEx

基本概念

RegOpenKeyEx是Windows注册表API中用于打开已存在注册表项的 增强版函数，在RegOpenKey的基础上新增了自定义访问权限和打开选项的支持，功能更灵活、安全，是开发中推荐优先使用的 “打开注册表项”API（尤其适合需要精细控制权限的场景）。

基本语法

返回值（与 RegOpenKey 一致）
成功：ERROR_SUCCESS（0）；
失败：非0错误码（如ERROR_FILE_NOT_FOUND项不存在、ERROR_ACCESS_DENIED权限不足、ERROR_INVALID_PARAMETER参数无效等），可通过FormatMessage解析错误描述。
